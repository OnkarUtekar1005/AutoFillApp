# Filing the AOC-4 form (Claude browser extension)

We fill the MCA V3 AOC-4 web form using the **Claude browser extension** driving
your **own logged-in Chrome** — not Playwright. This is simpler and more robust:

- **Login is free** — the form is filled in your real Chrome, already signed in to MCA.
- **No brittle selectors** — Claude locates each field by its on-screen label.
- **You stay in control** — Claude fills + verifies field-by-field and **stops before
  DSC signing and final submit**, which you always do yourself.

`filing/aoc4_form.py` (the old Playwright launcher) is **deprecated** and no longer
invoked by the app. It's kept only for reference.

## How to fill a client's form

1. In the dashboard, extract the client so its data is ready (green/amber status).
   Fill any MANUAL fields in the Manual Entry tab.
2. Click **Prepare MCA fill** on that client — the dashboard confirms how many
   fields are ready.
3. Open the **AOC-4 form on the MCA V3 portal** in Chrome and log in.
4. In a Claude session (with the browser extension connected), say:
   *"Fill \<Client Name\> on the MCA form."*
5. Claude reads the client's values from `GET /api/clients/<id>/fill-data`
   (label → value, in form order) and fills each field, verifying as it goes.
   It pauses at the "ready to sign" step and hands back to you for DSC + submit.

## The fill-data API

`GET /api/clients/<client_id>/fill-data` returns, per filing:

```json
{
  "client": { "id": "...", "name": "...", "cin": "...", "path": "..." },
  "filings": [{
    "client_name": "...", "cin": "...", "period": "...", "status": "Ready",
    "fields": [
      { "key": "cin", "label": "Corporate Identification Number (CIN)",
        "value": "1234354", "section": "Part A — Company Identity",
        "source": "AUTO" },
      ...
    ]
  }]
}
```

Only fields that have a value are included; MCA-generated fields (SRN, filing
date) are excluded. `source` is AUTO (extracted) or MANUAL (you entered).

## Test targets (safe rehearsal — not the real MCA portal)

Two mock forms to verify filling before touching MCA:

1. **Local mock form** — `http://localhost:8000/test-form` (also the dashboard
   **Test form** button). 271 labeled inputs; has an "Autofill from extracted
   data" button to sanity-check the data pipeline, and is a target for the
   extension to fill by label.

2. **Google Form** — generated by `filing/aoc4_test_form.gs` (Google Apps
   Script). Steps:
   - Open https://script.google.com → New project.
   - Paste the contents of `filing/aoc4_test_form.gs`, Save.
   - Run the function `createAOC4TestForm` (authorize it — it only creates a
     Form in your Drive).
   - Open Execution log → copy the **Live URL**. Open it in Chrome.
   - Ask Claude (extension connected) to fill it — the 31 question titles are
     exact AOC-4 field labels, so it's a realistic hosted-form test.

## Always manual (never automated)

- MCA login / OTP / captcha
- DSC signing (local emBridge/emSigner + physical token)
- Final submit
